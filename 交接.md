# 党员思想汇报生成器 - 项目交接文档

## 项目概述

这是一个**党员积极分子思想汇报生成工具**，面向中国投资促进会等机构的入党积极分子使用。

**在线访问**：https://junzhejin443-pixel.github.io/party-report-generator/

**当前版本**：v2.0（2026年7月全面重写）

## v2.0 重大改进

### 架构变化
- **删除快速模式路径**：统一为素材选择→生成流程，消除崩溃 bug
- **句式变体库替代词级替换**：每个语义位置预写 3-5 个完整通顺句子，消除不通顺问题
- **版本从零构建**：A/B/C 三版不再后处理变换，而是从模板层面生成结构不同的内容
- **LLM 集成**：可选用 Anthropic/OpenAI API 自动更新素材库和智能扩展工作描述

### Bug 修复（v2.0）
- 修复：中期模板崩溃（引用未定义变量 `工作详情`）
- 修复：A版/C版版本策略完全无效（死代码）
- 修复：字数调整在签名后追加文本
- 修复："此致敬礼"不符合官方格式
- 修复：第3条选中素材从未被引用

## 核心功能

### 1. 思想汇报生成
- 根据用户选择的年月，展示当月推荐的时政素材
- 支持选择3种风格：A版理论学习型、B版实践结合型、C版反思成长型
- 三个版本从结构层面不同（非仅标题差异），可生成2-3篇供选择
- 每类素材可选择1-3条，全部被引用到报告中

### 2. 素材库
- **时事新闻**：9条静态 + AI扩展（国内/国际）
- **领导讲话**：9条静态 + AI扩展（习近平重要讲话）
- **党的精神**：9条静态 + AI扩展（伟大建党精神、两个确立、四个自信等）

### 3. 时间递进机制
- 配置首次汇报年月和提交频率（月度/季度/半年）
- 自动计算第N次汇报，阶段递进：
  - 第1-2次：初期（2段式：学习体会 + 努力方向，简单词汇，高引用率）
  - 第3-4次：中期（3段式：理论认识 + 工作思考 + 不足改进，中等词汇）
  - 第5次+：后期（4段式：深入理解 + 工作反思 + 路径思考 + 提升计划，独立思考）
- 视觉进度指示器：三阶段节点+连线，当前阶段高亮

### 4. 去AI化（v2.0 句级变体）
- **句级变体库**（`句式库`）：~100句预写的自然中文句子
  - 3种开头风格 × 各4个变体
  - 3个阶段 × 各3个时政学习句式
  - 3个阶段 × 各3个讲话引用句式
  - 3个阶段 × 各3个精神引用句式
  - 6个过渡句 + 3个阶段结尾段
- 过渡句在段落间随机插入（60%概率），增加结构多样性

### 5. 工作内容智能扩展
- **LLM优先**：调用大模型语义理解用户输入，生成与当月主题关联的描述
- **本地降级**：改进的关键词精确映射（避免"国际"等过于宽泛的匹配）

### 6. AI素材更新（可选）
- 支持 Anthropic (Claude) 或 OpenAI (GPT) API
- 一键生成 5条新新闻 + 5条新讲话 + 5条新精神提法
- 与静态库智能去重合并
- API密钥仅存储本地浏览器

## 技术架构

### 单文件HTML应用
- 无后端，纯前端
- 所有JS/CSS/HTML在`index.html`中
- 数据存储在浏览器localStorage
- LLM调用通过浏览器fetch直接请求API

### 文件结构
```
party_report_generator.html  ← GitHub Pages入口（同步自index.html）
index.html              ← 开发主文件（两文件内容相同，v2.0 ~1636行）
交接.md                  ← 本文档
```

## 官方写作规范

根据[共产党员网范文](https://fuwu.12371.cn/2016/08/19/ARTI1471596869770792.shtml)：

### 格式要求
- 标题：居中写"思想汇报"
- 称谓："敬爱的党组织"
- 结尾："恳请党组织给予批评、帮助"
- 署名："汇报人：XXX" + "XXXX年XX月XX日"

### 内容要求
1. 向党组织如实汇报，敢于亮思想
2. 围绕一两个重点问题谈深谈透，不要罗列多个泛泛而谈
3. 一分为二评价自己，肯定进步，指出不足

## 维护指南（v2.0）

### 添加新的句式变体
在`句式库`对应分类中添加新的句子字符串：
```javascript
// 例如：添加一个新的A版开头
句式库.开头.A.push('新的理论型开头句式...');
```

### 添加新的精神提法
在`党的精神提法库`数组中添加对象：
```javascript
{ id: 'spirit10', 名称: '新提法名称', 描述: '具体描述', 推荐月份: [3, 10] }
```

### 添加新的领导讲话
在`领导讲话库`数组中添加：
```javascript
{ id: 'speech10', 来源: '习近平', 讲话: '讲话内容', 推荐月份: [6] }
```

### 修改模板结构
在`生成初期模板()` / `生成中期模板()` / `生成后期模板()` 函数中修改段落结构。
每个模板接受`(素材, 人物, 阶段, 版本, 工作详情)`参数，`版本`参数为'A'/'B'/'C'用于区分风格。

### 调整字数设置
修改`生成扩充段落()`中的初期/中期/后期扩充素材数组

### 调整阶段递进规则
在`getStageInfo()`函数中修改`reportNumber`与阶段的对应关系

## 部署说明

### GitHub Pages
1. 推送到GitHub仓库
2. Settings → Pages → Source: master branch, / (root)
3. 等待构建后访问 `https://用户名.github.io/仓库名/`

### 注意事项
- `index.html`和`party_report_generator.html`需要保持同步
- 提交前执行`cp index.html party_report_generator.html`
- v2.0 localStorage key 改为 `party_report_data_v2`（与旧版不冲突）
- LLM API密钥单独存储于 `llm_api_key_v2` 和 `llm_provider_v2`

## 问题排查

### 按钮无反应
- 检查浏览器控制台是否有JS错误（F12 → Console）
- 确保已填写姓名和单位信息或加载已有配置

### 生成内容单一
- 确保三种风格（A/B/C）都已生成
- 使用"AI更新素材库"扩充素材多样性
- 尝试选择不同的素材组合

### AI功能不可用
- 确认已在"AI助手设置"中配置API密钥
- 检查API密钥是否有效（使用"测试连接"按钮）
- 不配置API密钥不影响基本生成功能

### 旧版数据迁移
- v2.0 使用新的 localStorage key `party_report_data_v2`
- 旧版数据（key `party_report_data`）需手动重新填写配置
